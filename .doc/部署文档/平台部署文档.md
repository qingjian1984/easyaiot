## Overview

EasyAIoT is deployed via **Docker containers and a unified install script**. The platform consists of base middleware and business modules: DEVICE, AI, VIDEO, WEB, and APP.

| Module | Directory | Description |
|--------|-----------|-------------|
| Base services | `.scripts/docker` | Nacos, PostgreSQL, Redis, Kafka, MinIO, etc. |
| DEVICE | `DEVICE/` | Device management & API gateway (Java / Spring Cloud) |
| AI | `AI/` | Model training & inference (Python) |
| VIDEO | `VIDEO/` | Video streaming, alerts, recording (Python) |
| WEB | `WEB/` | Management console (Vue 3) |
| APP | `APP/` | Mobile H5 (**full** profile only) |

**Unified entry scripts** (Linux x86 examples below):

| OS | Script |
|----|--------|
| Linux x86 | `.scripts/docker/install_linux.sh` |
| Linux ARM | `.scripts/docker/install_linux_arm.sh` |
| Kylin | `.scripts/docker/install_linux_kylin.sh` |
| macOS | `.scripts/docker/install_mac.sh` |
| Windows | `.scripts/docker/install_win.ps1` |

## Two Usage Modes

The unified entry script supports **interactive guidance** and **direct command** modes with identical underlying capabilities:

| | Interactive | Direct Command |
|---|---|---|
| **Entry** | No args / `menu` / `interactive` | `<command> [args]` |
| **Use case** | First deploy, on-site ops, troubleshooting | Dev, scripted ops, CI/CD |
| **Operation** | Menu-driven, numeric selection | Execute subcommand directly |
| **After execution** | Returns to current menu level | Exits when done |

```bash
# Interactive
sudo .scripts/docker/install_linux.sh

# Direct command
sudo .scripts/docker/install_linux.sh install
.scripts/docker/install_linux.sh status
```

**Selection guide:**

- Manual day-to-day ops, unfamiliar with command args → Interactive
- Known operation, scripting or cron jobs → Direct command (**do not** invoke without args in Cron/CI — it will block waiting for input)

### Interactive: Menu Structure

**Root menu**

```
  1) Deploy — install, start/stop, update, status, logs
  2) Analyze — log merge, disk usage, health checks
  0) Exit
```

**[Deploy] submenu**

| # | Action | Equivalent command |
|:-:|--------|-------------------|
| 1 | First install & start | `install` |
| 2 | Start all services | `start` |
| 3 | Stop all services | `stop` |
| 4 | Restart all services | `restart` |
| 5 | View status | `status` |
| 6 | View logs | `logs` |
| 7 | Health verification | `verify` |
| 8 | Update images & restart | `update` |
| 9 | Check Docker environment | `check` |
| 10 | View deploy profile | `profile` |
| 11 | Full CLI help | `help` |

**[Analyze] submenu**

| # | Action | Equivalent command |
|:-:|--------|-------------------|
| 1 | Multi-module log merge (~500 lines per source) | `analyze-logs` |
| 2 | Disk usage analysis | `analyze-disk` |
| 3 | Status + health verification | `status` + `verify` |
| 4 | Docker environment check | `check` |

**Typical paths:**

| Scenario | Interactive path |
|----------|-----------------|
| First deployment | 1 → 1 → 7 |
| Start after reboot | 1 → 2 → 7 |
| Collect diagnostics | 2 → 3 → 1 → 2 |

## Quick Start

### Prerequisites

- OS: **Ubuntu 24.04+** (26.04 recommended)
- Docker + Docker Compose **v2.35+**
- **≥ 300 GB** free disk space

```bash
docker --version && docker compose version && docker ps
```

### Option 1: Interactive

```bash
git clone https://gitee.com/volara/easyaiot.git
cd easyaiot

sudo .scripts/docker/install_linux.sh
# 1 Deploy → 1 First install → 7 Health verify
```

Profile is selected interactively on first install. Open `http://<server-ip>:8888` when done.

### Option 2: Direct Command

```bash
git clone https://gitee.com/volara/easyaiot.git
cd easyaiot

# Optional: pull pre-built images to shorten install time
sudo .scripts/docker/install_linux.sh pull

sudo .scripts/docker/install_linux.sh install
.scripts/docker/install_linux.sh verify
```

### Install Duration

| Scenario | Estimated time |
|----------|---------------|
| Pre-built images pulled | 10–30 minutes |
| Full local build | 30 minutes to several hours |

`install` flow: select profile → env check → create network → deploy middleware & modules → health wait. See [One-Click & Step-by-Step Deployment](./部署最佳实践_en.md#one-click--step-by-step-deployment).

## Deployment Profiles

Selected interactively on first `install`, saved to `.scripts/docker/.deploy_profile`. Reused by subsequent `start` / `stop` / `update`.

| Option | Name | Recommended RAM | Use case |
|:------:|------|-----------------|----------|
| 1 | **mini** | ≥ 4 GB | Edge nodes, PoC |
| 2 | **standard** | ≥ 16 GB | Regular production |
| 3 | **full** (default) | ≥ 20 GB | Full features + APP H5 |

```bash
.scripts/docker/install_linux.sh profile                              # View current profile
export EASYAIOT_DEPLOY_PROFILE=full && sudo .../install_linux.sh install  # Non-interactive
```

Profile service differences: [Deployment Profile Selection](./部署最佳实践_en.md#deployment-profile-selection).

## Script Command Reference

### Commands

| Command | Description |
|---------|-------------|
| `install` | First install & start |
| `start` / `stop` / `restart` | Lifecycle control |
| `status` | View running status |
| `logs [module]` | View logs, e.g. `logs VIDEO` |
| `verify` | Health check |
| `check` | Docker environment check |
| `update` | Update images & restart |
| `pull` | Pull pre-built images |
| `build` | Rebuild images locally |
| `profile` | View deploy profile |
| `analyze-logs` | Multi-module log merge |
| `analyze-disk` | Disk usage analysis |
| `diagnose` | Enter [Analyze] submenu |
| `clean` | Remove containers & images ⚠️ (includes volumes) |
| `help` | Show help |
| `menu` | Open interactive guide |

### Non-Interactive Log Collection

```bash
cd .scripts/docker

./analyze_merge_logs.sh --non-interactive \
  --modules dev-iot-sink,dev-iot-message,biz-video --lines 500 --save

./analyze_merge_logs.sh --non-interactive --modules DEVICE --save
./analyze_disk_usage.sh --save --top 15
```

### Mode Cross-Reference

| Action | Interactive | Direct command |
|--------|-------------|----------------|
| First install | 1 → 1 | `install` |
| Start services | 1 → 2 | `start` |
| Health check | 1 → 7 | `verify` |
| Log merge | 2 → 1 | `analyze-logs` |
| Disk analysis | 2 → 2 | `analyze-disk` |

### Per-Module Deployment

```bash
cd .scripts/docker && ./install_middleware_linux.sh install   # Middleware only
cd .scripts/docker && ./install_business_linux.sh install     # Business modules only
cd AI && ./install_linux.sh install                           # Single module
```

## FAQ

| Symptom | Resolution |
|---------|------------|
| Docker `permission denied` | `sudo usermod -aG docker $USER && newgrp docker` |
| Compose version too old | `sudo apt install -y docker-compose-plugin` |
| Port in use | `ss -tlnp \| grep <port>` |
| Install failed | `tail .scripts/docker/logs/install_linux_*.log` |
| Services up but unreachable | `verify` + check firewall |
| Insufficient disk | `df -h /`, reserve ≥ 300 GB |

**Collect diagnostics:**

```bash
# Interactive: 2 Analyze → 1 Logs + 2 Disk
# Direct command:
.scripts/docker/install_linux.sh check
.scripts/docker/install_linux.sh status
.scripts/docker/install_linux.sh verify
cd .scripts/docker && ./analyze_merge_logs.sh --non-interactive --modules all --save
./analyze_disk_usage.sh --save
```

More: [Troubleshooting](./部署最佳实践_en.md#troubleshooting).

## Environment Requirements

| Item | Requirement |
|------|-------------|
| OS | Ubuntu 24.04+ (26.04 recommended); also macOS, Windows, ARM, Kylin |
| CPU | Min. 4 cores, 8+ recommended |
| RAM | Depends on profile (full ≥ 20 GB, 32 GB recommended) |
| Disk | Min. 300 GB free, 500 GB+ SSD recommended |
| GPU | Optional; NVIDIA GPU (CUDA 12.8) for AI training/inference |
| Docker Compose | v2.35.0+ |

```bash
# Docker install (Ubuntu)
curl -fsSL https://get.docker.com | sudo sh
sudo apt install -y docker-compose-plugin
sudo usermod -aG docker $USER && newgrp docker
```

**Notes:**

1. Use `sudo` on first install (mirror acceleration & RTP port reservation)
2. Change default middleware passwords in production ([credentials](./部署最佳实践_en.md#default-credentials))
3. `clean` deletes volumes — back up first
4. Rebuild WEB after profile change: `cd WEB && ./install_linux.sh build`
