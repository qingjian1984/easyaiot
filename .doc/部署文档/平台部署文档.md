## Overview

EasyAIoT is a cloud-edge integrated intelligent algorithm platform deployed with **Docker containers and a unified install script**.

### Platform Components

| Module | Directory | Description |
|--------|-----------|-------------|
| Base services | `.scripts/docker` | Nacos, PostgreSQL, Redis, Kafka, MinIO, etc. |
| DEVICE | `DEVICE/` | Device management & API gateway (Java / Spring Cloud) |
| AI | `AI/` | Model training, inference, OCR, LLM (Python) |
| VIDEO | `VIDEO/` | Video streaming, alerts, recording, face recognition (Python) |
| WEB | `WEB/` | Management console (Vue 3) |
| APP | `APP/` | Mobile H5 (**full** profile only) |

### Unified Entry Scripts

| OS | Script |
|----|--------|
| Linux | `.scripts/docker/install_linux.sh` |
| macOS | `.scripts/docker/install_mac.sh` |
| Windows | `.scripts/docker/install_win.ps1` |

> Examples below use **Linux**; replace `install_linux.sh` with the matching script on macOS/Windows.

## Environment Requirements

### System & Hardware

| Item | Requirement |
|------|-------------|
| **OS** | **Ubuntu 24.04 LTS or newer** (**Ubuntu 26.04 LTS recommended**); also macOS 10.15+, Windows 10/11 |
| **CPU** | Min. 4 cores, 8+ recommended |
| **RAM** | Depends on profile; full profile min. 20 GB, 32 GB recommended |
| **Disk** | **Min. 300 GB free**, 500 GB+ SSD recommended |
| **GPU** | Optional; NVIDIA GPU (CUDA 12.8) recommended for AI |

### Software Dependencies

| Software | Version | Verify |
|----------|---------|--------|
| Docker | Installed, daemon accessible | `docker --version` |
| Docker Compose | **v2.35.0+** (`docker compose` plugin) | `docker compose version` |
| curl | Health checks | `curl --version` |

Install on Ubuntu:

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo apt install -y docker-compose-plugin
sudo usermod -aG docker $USER && newgrp docker
docker ps
```

### Docker Permissions (Linux)

```bash
sudo usermod -aG docker $USER
newgrp docker          # or log in again
docker ps              # should succeed without permission denied
```

Use `sudo` for first install so the script can configure mirror acceleration and RTP port reservation.

## Quick Start

### Linux: Four Steps

```bash
# ① Clone
git clone https://gitee.com/volara/easyaiot.git
cd easyaiot

# ② Self-check (optional but recommended)
.scripts/docker/install_linux.sh check
.scripts/docker/detect_system_info.sh

# ③ One-click install (profile selection 1/2/3 on first run)
sudo .scripts/docker/install_linux.sh install

# ④ Verify and open browser
.scripts/docker/install_linux.sh verify
# http://<server-ip>:8888
```

### What Happens During Install?

1. Select **deployment profile** (mini / standard / full)
2. Check Docker, Compose, container creation
3. Detect host IP, create `easyaiot-network`
4. Deploy in order: middleware → DEVICE → AI → VIDEO → WEB → APP (full)
5. Print service URLs

**Estimated duration**:

- With pre-built images: **~10–30 minutes**
- Full local build: **30 minutes to several hours**

To shorten install: run `.scripts/docker/install_linux.sh pull` before install (see [Pre-Built Images](./部署最佳实践_en.md#pre-built-images-optional)).

### macOS Quick Start

```bash
git clone https://gitee.com/volara/easyaiot.git && cd easyaiot
cd .scripts/docker && chmod +x install_mac.sh
./install_mac.sh install
./install_mac.sh verify
```

## Deployment Profiles

On first `install`, you choose a profile interactively. It is saved to `.scripts/docker/.deploy_profile`.

| Option | Name | Recommended RAM | Typical Use |
|:------:|------|-----------------|-------------|
| 1 | **mini** | ≥ 4 GB | Edge nodes, PoC |
| 2 | **standard** | ≥ 16 GB | Regular production (no TDengine/EMQX, etc.) |
| 3 | **full** (default) | ≥ 20 GB | Full features + APP H5 |

View current profile:

```bash
.scripts/docker/install_linux.sh profile
```

Non-interactive:

```bash
export EASYAIOT_DEPLOY_PROFILE=full
sudo .scripts/docker/install_linux.sh install
```

Profile differences: [Deployment Best Practices — Profile Selection](./部署最佳实践_en.md#deployment-profile-selection).

## Script Reference

### Commands

| Command | Description | Example |
|---------|-------------|---------|
| `install` | First install & start | `./install_linux.sh install` |
| `start` | Start all services | `./install_linux.sh start` |
| `stop` | Stop all services | `./install_linux.sh stop` |
| `restart` | Restart all | `./install_linux.sh restart` |
| `status` | Show status | `./install_linux.sh status` |
| `logs` | View logs | `./install_linux.sh logs` |
| `logs <module>` | Module logs | `./install_linux.sh logs VIDEO` |
| `build` | Rebuild images locally | `./install_linux.sh build` |
| `pull` | Pull pre-built images | `./install_linux.sh pull` |
| `update` | Update & restart | `./install_linux.sh update` |
| `verify` | Health check | `./install_linux.sh verify` |
| `check` | Check Docker env | `./install_linux.sh check` |
| `profile` | Show deploy profile | `./install_linux.sh profile` |
| `clean` | Remove containers & images ⚠️ | `./install_linux.sh clean` |
| `help` | Show help | `./install_linux.sh help` |

> From project root, use `.scripts/docker/install_linux.sh` instead of `./install_linux.sh`.

### `install`

First deployment; installs enabled modules in dependency order.

```bash
sudo .scripts/docker/install_linux.sh install
```

### `verify`

Checks ports and health endpoints; prints URLs on success:

```
[SUCCESS] All services are running!

Service URLs:
  Middleware (Nacos):     http://localhost:8848/nacos
  Middleware (MinIO):     http://localhost:9000 (API), http://localhost:9001 (Console)
  DEVICE (Gateway):       http://localhost:48080
  AI:                     http://localhost:5000
  VIDEO:                  http://localhost:6000
  WEB:                    http://localhost:8888
```

### `clean` ⚠️

**Dangerous**: removes containers, images, and volumes. Requires confirmation (`y`).

### Per-Module / Business-Only Deploy

```bash
# Middleware only
cd .scripts/docker && ./install_middleware_linux.sh install

# Business modules only
cd .scripts/docker && ./install_business_linux.sh install

# Single module (e.g. AI)
cd AI && ./install_linux.sh install
```

## Module Overview

### Base Services (`.scripts/docker`)

Middleware managed by `install_middleware_linux.sh`: Nacos, PostgreSQL, Redis, TDengine, Kafka, MinIO, Milvus, SRS, EMQX, ZLMediaKit, Node-RED, etc. (actual set depends on profile).

### DEVICE

- **Stack**: Java 21, Spring Boot 2.7, Spring Cloud Gateway
- **Features**: Device access, products, rules, GB28181, system admin
- **Port**: 48080 (Gateway)

### AI

- **Stack**: Flask, PyTorch 2.9+ (CUDA 12.8)
- **Features**: Train/infer/deploy models, OCR, speech, LLM
- **Port**: 5000

### VIDEO

- **Stack**: Flask, OpenCV, FFmpeg
- **Features**: Streaming, real-time/snapshot algorithms, recording, alerts, face recognition
- **Port**: 6000

### WEB

- **Stack**: Vue 3.4, TypeScript, Vite, Ant Design Vue 4
- **Port**: 8888

### APP (full only)

- **Description**: Mobile H5
- **Port**: 9010

## FAQ

### 1. Docker Permission Denied

```bash
sudo usermod -aG docker $USER
newgrp docker
docker ps
```

### 2. Docker Compose Too Old

Requires **v2.35.0+**:

```bash
sudo apt update && sudo apt install -y docker-compose-plugin
docker compose version
```

### 3. Port In Use

```bash
ss -tlnp | grep <port>
# Stop the process or edit docker-compose.yml port mapping
```

### 4. Install Failed Midway

```bash
ls -lt .scripts/docker/logs/ | head -5
tail -100 .scripts/docker/logs/install_linux_*.log
docker ps -a
.scripts/docker/install_linux.sh status
```

### 5. Services Up but Browser Cannot Connect

```bash
.scripts/docker/install_linux.sh verify
sudo ufw allow 8888    # if firewall enabled
.scripts/docker/install_linux.sh logs WEB
```

### 6. Insufficient Disk Space

First build uses a lot of disk—**reserve ≥ 300 GB**:

```bash
df -h /
docker system df
.scripts/docker/cleanup_docker_space.sh
```

### 7. WEB Broken After Profile Change

WEB image is tied to deploy profile—rebuild after switching:

```bash
cd WEB && ./install_linux.sh build
```

More: [Troubleshooting](./部署最佳实践_en.md#troubleshooting).

## Deployment Workflow

### First-Deploy Checklist

- [ ] Ubuntu ≥ 24.04, ≥ 300 GB disk free
- [ ] Docker + Compose v2.35+ installed
- [ ] `docker ps` works for current user
- [ ] Core ports free
- [ ] Profile chosen (mini / standard / full)
- [ ] Run `install` → `verify` → open `:8888`

### Daily Operations

```bash
.scripts/docker/install_linux.sh start
.scripts/docker/install_linux.sh status
.scripts/docker/install_linux.sh logs
.scripts/docker/install_linux.sh restart
```

### Updates

```bash
git pull
sudo .scripts/docker/install_linux.sh update
.scripts/docker/install_linux.sh verify
```

## Notes

1. **Profiles**: Match RAM to profile; use `analyze_deploy_memory.sh` to analyze
2. **Disk**: Local builds and volumes grow fast—**min. 300 GB**, 500 GB+ SSD for production
3. **sudo**: Recommended on first install for mirror and RTP setup
4. **Passwords**: Change default middleware passwords in production ([credentials](./部署最佳实践_en.md#default-credentials))
5. **clean**: Deletes volumes—back up first
6. **Network**: Access to Docker Hub or configured mirror required
