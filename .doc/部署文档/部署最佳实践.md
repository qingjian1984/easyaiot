## 两种使用模式（详细）

统一入口脚本（`install_linux.sh` / `install_linux_arm.sh` / `install_linux_kylin.sh`）支持 **两种等价用法**：

| 模式 | 入口 | 受众 | 特点 |
|------|------|------|------|
| **交互引导** | 无参数 / `menu` / `interactive` | 现场运维、手动操作 | 中文菜单、分步引导、执行后自动回到当前菜单 |
| **指定命令** | `<命令> [参数]` | 开发、SRE、CI/CD | 可脚本化、可重复、执行完即退出 |

```bash
# 交互引导
sudo .scripts/docker/install_linux.sh

# 指定命令
sudo .scripts/docker/install_linux.sh install
.scripts/docker/install_linux.sh status
```

**选型建议：** 手动运维优先交互引导；脚本化场景（Cron / Ansible / CI）必须使用指定命令，禁止无参数调用。

### 交互引导：菜单结构

**根菜单**

```
  1) 部署 — 安装、启动、停止、更新等服务操作
  2) 分析 — 日志、磁盘、状态等问题定位
  0) 退出
```

**【部署】子菜单**

| # | 操作 | 等价命令 |
|:-:|------|----------|
| 1 | 首次安装并启动 | `install` |
| 2 | 启动所有服务 | `start` |
| 3 | 停止所有服务 | `stop` |
| 4 | 重启所有服务 | `restart` |
| 5 | 查看运行状态 | `status` |
| 6 | 查看服务日志 | `logs` |
| 7 | 验证服务健康 | `verify` |
| 8 | 更新镜像并重启 | `update` |
| 9 | 检查 Docker 环境 | `check` |
| 10 | 查看部署规格 | `profile` |
| 11 | 完整命令行帮助 | `help` |

**【分析】子菜单** — 输出建议直接发给技术支持

| # | 操作 | 等价命令 |
|:-:|------|----------|
| 1 | 多模块日志合并 | `analyze-logs` |
| 2 | 磁盘占用分析 | `analyze-disk` |
| 3 | 状态 + 健康验证 | `status` + `verify` |
| 4 | Docker 环境检查 | `check` |

**日志合并内层**（从分析 → 1 进入）：按序号多选日志源（如 `24,23,27`），`0` = 当前规格下全部，`b` = 返回【分析】。

### 指定命令：完整参考

```bash
cd .scripts/docker   # 以下均可用 .scripts/docker/install_linux.sh 从项目根执行

# 生命周期
./install_linux.sh install | start | stop | restart | update | clean

# 可观测性
./install_linux.sh status | logs | logs WEB | verify | check | profile

# 构建与镜像
./install_linux.sh build | pull | build-runtime [模块]

# 问题分析
./install_linux.sh diagnose          # 进入【分析】子菜单（仍交互）
./install_linux.sh analyze-logs      # 日志合并
./install_linux.sh analyze-disk      # 磁盘报告

# 帮助
./install_linux.sh help | menu
```

### 分析工具：开发者进阶

分析脚本位于 `.scripts/docker/`，可独立执行：

**多模块日志合并 `analyze_merge_logs.sh`**

```bash
cd .scripts/docker

# 非交互（推荐写入 runbook）
./analyze_merge_logs.sh --non-interactive \
  --modules dev-iot-sink,dev-iot-message,biz-video --lines 500 --save

# 模块别名
./analyze_merge_logs.sh --non-interactive --modules DEVICE
./analyze_merge_logs.sh --non-interactive --modules .scripts/docker
./analyze_merge_logs.sh --non-interactive --modules all --save

# 常用单元 ID：mw-nacos / mw-postgres / dev-iot-gateway / dev-iot-sink / biz-ai / biz-video / biz-web
./analyze_merge_logs.sh --help
```

采集策略：优先 `docker logs`（最近 N 行）→ 容器不可用时读宿主机日志 → 滚动日志取最新文件尾部。

**磁盘占用 `analyze_disk_usage.sh`**

```bash
./analyze_disk_usage.sh                  # 终端报告
./analyze_disk_usage.sh --save           # 落盘到 logs/disk_usage_*.log
./analyze_disk_usage.sh --top 20
```

重点目录：MinIO `record-space` / `alert-images`、本地 `playbacks`、告警图中转目录。

### 自动化注意事项

- Cron / Ansible / CI **禁止**无参数调用（会阻塞在菜单）
- 从菜单内部触发的操作设 `EASYAIOT_FROM_MENU=1`，避免 install 后弹回根菜单；命令行模式无此行为
- 非交互指定规格：`export EASYAIOT_DEPLOY_PROFILE=full`

### 与单模块脚本的关系

各模块目录（`DEVICE/`、`AI/`、`VIDEO/` …）有独立 `install_linux.sh`，仅管理该模块，**不含**统一【分析】菜单。  
全平台编排 + 交互引导 + 跨模块日志/磁盘分析 → 只用 `.scripts/docker/install_linux.sh`。

## 5 分钟部署流程

```bash
git clone https://gitee.com/volara/easyaiot.git && cd easyaiot

docker --version && docker compose version

# 方式 A：指定命令
sudo .scripts/docker/install_linux.sh pull    # 可选，拉预构建镜像
sudo .scripts/docker/install_linux.sh install
.scripts/docker/install_linux.sh verify

# 方式 B：交互引导
sudo .scripts/docker/install_linux.sh         # 1 部署 → 1 安装 → 7 验证

# 访问：http://<服务器IP>:8888
```

### 安装耗时

| 情况 | 耗时 |
|------|------|
| 已拉预构建镜像 | 10～30 分钟 |
| 本地完整构建 | 30 分钟～数小时 |

## 部署规格选型

首次 `install` 交互选择，或 `export EASYAIOT_DEPLOY_PROFILE=mini|standard|full`。  
结果保存在 `.scripts/docker/.deploy_profile`，后续 `start` / `stop` / `update` 自动沿用。

| 规格 | 别名 | 推荐内存 | 适用场景 |
|------|------|----------|----------|
| **mini** | `1` / `4g` | ≥ 4 GB | 边缘节点、PoC |
| **standard** | `2` / `16g` | ≥ 16 GB | 常规生产 |
| **full** | `3`（默认） | ≥ 20 GB | 完整功能 + APP H5 |

```bash
.scripts/docker/install_linux.sh profile
```

### 各规格包含的服务

**mini**

- 业务：`iot-system`、VIDEO、AI、WEB
- 中间件：PostgreSQL、Redis、SRS
- 不启动：Nacos、Gateway、Kafka、iot-sink、MinIO、Milvus、ZLMediaKit、Node-RED、TDengine、EMQX 及多数 DEVICE 子模块
- API 路由：nginx 将 `/admin-api`、`/dev-api` 直连 `iot-system:48099`

**standard**

- 不启动：TDengine、EMQX、Node-RED、`iot-device`、`iot-tdengine`
- 其余全部启动

**full**

- 全部业务模块与中间件，含 **APP 移动端 H5**（9010）

内存分析：

```bash
.scripts/docker/analyze_deploy_memory.sh
.scripts/docker/analyze_deploy_memory.sh --all-profiles
```

## 环境要求与部署前检查

### 硬件

| 资源 | 最低 | 推荐 |
|------|------|------|
| CPU | 4 核 | 8 核+ |
| 内存 | 见部署规格（full ≥ 20 GB） | 32 GB+ |
| 磁盘 | **300 GB** 可用 | 500 GB+ SSD |
| GPU | 无（CPU 可运行） | NVIDIA GPU（CUDA 12.8） |

### 软件

| 软件 | 要求 |
|------|------|
| 操作系统 | Ubuntu 24.04+（建议 26.04）；亦支持银河麒麟、ARM64 |
| Docker | 已安装且 daemon 可访问 |
| Docker Compose | **v2.35.0+**（`docker compose` 插件） |
| NVIDIA Driver / Container Toolkit | 仅 GPU 场景 |

### Docker 权限

```bash
sudo usermod -aG docker $USER && newgrp docker
docker ps   # 应无 permission denied
```

首次安装建议 `sudo`，以配置镜像源与 RTP 端口预留。

### 部署前检查

```bash
.scripts/docker/detect_system_info.sh      # 系统信息
.scripts/docker/install_linux.sh check      # Docker 环境
df -h / && docker system df                  # 磁盘
```

## 一键部署与分步部署

### 一键部署

```bash
sudo .scripts/docker/install_linux.sh install
.scripts/docker/install_linux.sh verify
```

**install 自动流程：**

1. 选择部署规格 → 写入 `.deploy_profile`
2. 预构建镜像检测（若已 pull 则跳过本地 build）
3. Docker / Compose / 容器创建能力检查
4. 宿主机 IP 检测（可设 `HOST_IP=<ip>`）
5. RTP 端口 30000-30500 预留（需 root）
6. Docker 镜像加速配置（需 root）
7. 创建 `easyaiot-network`
8. 按序部署：中间件 → DEVICE → AI → VIDEO → WEB → APP（full）
9. 等待 PostgreSQL / Nacos / Redis 就绪
10. 按需确保边缘 Agent

### 分步部署

先确定规格：

```bash
export EASYAIOT_DEPLOY_PROFILE=full
```

**第一步：中间件**

```bash
cd .scripts/docker && ./install_middleware_linux.sh install
```

| 中间件 | 端口 | 用途 |
|--------|------|------|
| Nacos | 8848 | 注册/配置 |
| PostgreSQL | 5432 | 主数据库（6 库） |
| Redis | 6379 | 缓存 |
| Kafka | 9092 | 消息队列 |
| MinIO | 9000/9001 | 对象存储 |
| Milvus | 19530/9091 | 向量库 |
| SRS | 1935 | 流媒体 |
| EMQX | 1883 | MQTT（full） |
| ZLMediaKit | 6080 | 媒体服务器 |
| TDengine | 6030 | 时序库（full） |
| Node-RED | 1880 | 规则引擎 |

**第二步～：业务模块**

```bash
cd DEVICE && ./install_linux.sh install
cd AI    && ./install_linux.sh install
cd VIDEO && ./install_linux.sh install
cd WEB   && ./install_linux.sh install
cd APP   && ./install_linux.sh install   # 仅 full
```

**仅业务模块（不含中间件）**

```bash
cd .scripts/docker
./install_business_linux.sh install
./install_business_linux.sh update DEVICE WEB
./install_business_linux.sh verify
```

## 常用运维命令

### 统一脚本

```bash
./install_linux.sh install | start | stop | restart | status
./install_linux.sh logs | logs WEB | verify | check | profile
./install_linux.sh build | pull | update | clean
./install_linux.sh diagnose | analyze-logs | analyze-disk | help
```

### 单模块脚本

每个模块（`DEVICE` / `AI` / `VIDEO` / `WEB` / `APP`）：

```bash
./install_linux.sh install | start | stop | restart | status | logs | build | clean | update
```

中间件：

```bash
cd .scripts/docker
./install_middleware_linux.sh install | start | stop | restart | status | logs | build | clean | update
```

### 常用环境变量

| 变量 | 说明 |
|------|------|
| `EASYAIOT_DEPLOY_PROFILE` | `mini` / `standard` / `full` |
| `HOST_IP` | 强制宿主机 IP |
| `PARALLEL_MODULES=true` | 业务模块并行启动/更新 |
| `PARALLEL_BUILD=true` | 并行构建（默认串行防 OOM） |
| `FORCE_NETWORK_RECREATE=true` | IP 变更后重建网络 |
| `EASYAIOT_RUNTIME_REGISTRY` | 预构建镜像仓库 |

## 预构建镜像

配置文件：`.scripts/docker/runtime_registry.conf`

```bash
.scripts/docker/install_linux.sh pull                    # 交互拉取
.scripts/docker/install_linux.sh build-runtime           # 构建并推送（CI/发布）
.scripts/docker/install_linux.sh build-runtime DEVICE    # 单模块
```

拉取成功后，`install` / `update` 检测到 `.runtime_images_pulled` 标记，直接启动容器。

## 故障排查

### 推荐流程

**交互引导：**

```
无参数 → 2 分析 → 4 Docker 检查 → 3 状态+健康 → 1 日志 → 2 磁盘
```

**指定命令：**

```bash
.scripts/docker/install_linux.sh check
.scripts/docker/install_linux.sh status
.scripts/docker/install_linux.sh verify

cd .scripts/docker
./analyze_disk_usage.sh --save
./analyze_merge_logs.sh --non-interactive --modules dev-iot-sink,biz-video,mw-nacos --lines 500 --save
```

### 常见问题

**服务启动失败**

```bash
docker ps -a
docker logs -f postgres-server
.scripts/docker/install_linux.sh logs
```

**网络问题（宿主机 IP 变更）**

```bash
export FORCE_NETWORK_RECREATE=true
.scripts/docker/install_linux.sh restart
```

**PostgreSQL / Redis**

```bash
.scripts/docker/fix_postgresql.sh
.scripts/docker/fix_redis.sh
```

**Docker 系统问题**

```bash
sudo .scripts/docker/diagnose_docker_systemd.sh diagnose
.scripts/docker/cleanup_docker_space.sh
```

**Kafka 消费组**

```bash
cd VIDEO && ./fix_kafka_consumer_group.sh
```

**切换规格后 WEB 异常**

```bash
cd WEB && ./install_linux.sh build
```
