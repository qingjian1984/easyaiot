## 5 分钟快速上手

```bash
# 1. 获取代码
git clone https://gitee.com/volara/easyaiot.git
cd easyaiot

# 2. 确认 Docker 可用（见下文「环境要求」）
docker --version
docker compose version

# 3. 一键安装（首次会交互选择部署规格；建议用 sudo 以配置镜像源与 RTP 端口）
sudo .scripts/docker/install_linux.sh install

# 4. 验证
.scripts/docker/install_linux.sh verify

# 5. 浏览器访问管理平台
# http://<服务器IP>:8888
```

**首次安装耗时说明**：若未配置预构建镜像，脚本会在本地 `docker build` 各业务模块（DEVICE / AI / VIDEO / WEB），通常需要 **30 分钟～数小时**，取决于 CPU、磁盘与网络。可先执行 `pull` 拉取预构建镜像以显著缩短时间（见 [预构建镜像](#预构建镜像可选)）。

## 部署规格选型

安装时脚本会交互选择 **部署规格**（也可通过环境变量 `EASYAIOT_DEPLOY_PROFILE` 指定）。选择结果保存在 `.scripts/docker/.deploy_profile`，后续 `start` / `stop` / `update` 会自动沿用。

| 规格 | 别名 | 推荐内存 | 适用场景 |
|------|------|----------|----------|
| **mini** | `1` / `4g` | ≥ 4 GB | 边缘节点、功能验证、资源受限环境 |
| **standard** | `2` / `16g` | ≥ 16 GB | 常规生产，不含部分重型组件 |
| **full** | `3`（默认） | ≥ 20 GB | 完整功能，含 APP 移动端 H5 |

查看当前规格与服务范围：

```bash
.scripts/docker/install_linux.sh profile
```

### 各规格包含的服务

**mini（边缘精简版）**

- 业务：`iot-system`、VIDEO、AI、WEB
- 中间件：PostgreSQL、Redis、SRS
- 不启动：Nacos、Gateway、Kafka、iot-sink、MinIO、Milvus、ZLMediaKit、Node-RED、TDengine、EMQX 及多数 DEVICE 子模块
- API 路由：nginx 将 `/admin-api`、`/dev-api` 直连 `iot-system:48099`

**standard（标准版）**

- 不启动：TDengine、EMQX、Node-RED、`iot-device`、`iot-tdengine`
- 其余业务模块与中间件全部启动

**full（完整版）**

- 启动全部业务模块与中间件，含 **APP 移动端 H5**（端口 9010）

分析当前容器内存占用是否与规格匹配：

```bash
.scripts/docker/analyze_deploy_memory.sh
.scripts/docker/analyze_deploy_memory.sh --all-profiles   # 对比三种规格
```

## 环境要求

### 硬件要求

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 4 核 | 8 核+ |
| 内存 | 见 [部署规格选型](#部署规格选型)（full 最低 20 GB） | 32 GB+ |
| 磁盘 | **300 GB** 可用空间 | 500 GB+ SSD |
| GPU | 无（CPU 可运行） | NVIDIA GPU（CUDA 12.8，AI 推理/训练） |

> 磁盘主要用于：Docker 镜像层、构建缓存（`.build-cache/`）、数据库与对象存储数据卷。首次本地构建会占用大量空间，建议预留充足余量。

### 软件要求

| 软件 | 要求 | 说明 |
|------|------|------|
| 操作系统 | **Ubuntu 24.04 LTS**（最低） | **建议 Ubuntu 26.04 LTS**；亦支持银河麒麟、ARM64（见 [特殊环境](#特殊环境)） |
| Docker | 已安装且 daemon 可访问 | 未安装时：`curl -fsSL https://get.docker.com \| sudo sh` |
| Docker Compose | **v2.35.0+**（插件 `docker compose`） | 未安装时：`sudo apt install docker-compose-plugin` |
| NVIDIA Driver | 525+ | 仅 GPU 场景 |
| NVIDIA Container Toolkit | 最新版 | 仅 GPU 场景 |

### Docker 权限（Linux）

```bash
# 将当前用户加入 docker 组（推荐）
sudo usermod -aG docker $USER
newgrp docker   # 或重新登录

# 验证
docker ps
```

> 配置 Docker 镜像源、RTP 端口预留等系统级操作需要 root，**建议首次安装使用 `sudo` 运行**。

## 一键部署

### 入口脚本

统一编排脚本：`.scripts/docker/install_linux.sh`

```bash
# 从项目根目录执行（推荐）
sudo .scripts/docker/install_linux.sh install

# 或进入脚本目录
cd .scripts/docker
sudo ./install_linux.sh install
```

### install 自动执行的流程

1. **选择部署规格** — 交互选择 mini / standard / full，写入 `.deploy_profile`
2. **预构建镜像** — 若已配置远程仓库且选择拉取，跳过本地 build
3. **环境检查** — Docker、Docker Compose、容器创建能力（含 `/dev/null` 检测）
4. **宿主机 IP 检测** — 注入 GB28181 / ZLMediaKit 媒体地址（可设 `HOST_IP=<ip>` 跳过探测）
5. **RTP 端口预留** — 配置内核保留 30000-30500（需 root）
6. **Docker 镜像源** — 自动配置 `docker.m.daocloud.io` 加速（需 root）
7. **创建 Docker 网络** — `easyaiot-network`
8. **按依赖顺序部署模块**：
   - 中间件（`.scripts/docker/install_middleware_linux.sh`）
   - DEVICE → AI → VIDEO → WEB → APP（full）
9. **等待基础服务就绪** — PostgreSQL / Nacos / Redis 健康检查
10. **平台 Agent** — 按需确保边缘 Agent 可用

### 验证部署

```bash
.scripts/docker/install_linux.sh verify
```

成功输出示例：

```
服务访问地址:
  基础服务 (Nacos):     http://localhost:8848/nacos
  基础服务 (MinIO):     http://localhost:9000 (API), http://localhost:9001 (Console)
  基础服务 (Milvus):    http://localhost:9091 (Health), localhost:19530 (gRPC)
  Device服务 (Gateway): http://localhost:48080
  AI服务:               http://localhost:5000
  Video服务:            http://localhost:6000
  Web前端:              http://localhost:8888
  App移动端H5:          http://localhost:9010    # 仅 full
```

浏览器打开 `http://<服务器IP>:8888` 访问管理平台。

## 分步部署

需要精细控制时，可按模块分步执行。**务必先确定并导出部署规格**，保证各模块配置一致：

```bash
export EASYAIOT_DEPLOY_PROFILE=full   # 或 mini / standard
```

### 第一步：中间件

```bash
cd .scripts/docker
./install_middleware_linux.sh install
```

| 中间件 | 镜像 | 端口 | 用途 |
|--------|------|------|------|
| Nacos | nacos/nacos-server:v2.5.1 | 8848 | 服务注册与配置 |
| PostgreSQL | postgres:18 | 5432 | 主数据库（6 个业务库） |
| Redis | redis:7.4.8 | 6379 | 缓存 |
| Kafka | apache/kafka:3.8.0 | 9092 | 消息队列 |
| MinIO | minio/minio | 9000/9001 | 对象存储 |
| Milvus | milvusdb/milvus:v2.6.0 | 19530/9091 | 向量库（人脸识别） |
| SRS | ossrs/srs:5 | 1935 | 流媒体 |
| EMQX | emqx/emqx:5.8.7 | 1883 | MQTT（full profile） |
| ZLMediaKit | zlmediakit/zlmediakit:master | 6080 | 媒体服务器 |
| TDengine | tdengine/tsdb:3.3.8.4 | 6030 | 时序库（full profile） |
| Node-RED | nodered/node-red:latest | 1880 | 规则引擎 |

就绪检查：

```bash
docker exec postgres-server pg_isready -U postgres
curl -s http://localhost:8848/nacos/actuator/health
docker exec redis-server redis-cli -a basiclab@iot975248395 ping
```

### 第二步：DEVICE 服务

```bash
cd DEVICE
./install_linux.sh install
```

| 服务 | 端口 | 说明 |
|------|------|------|
| iot-gateway | 48080 | API 网关 |
| iot-system | 48099 | 系统管理 |
| iot-infra | 48066 | 基础设施 |
| iot-device | 48055 | 设备管理 |
| iot-dataset | 48077 | 数据集 |
| iot-message | 48033 | 消息推送 |
| iot-file | 48022 | 文件服务 |
| iot-sink | 48011 | 协议适配 |
| iot-gb28181 | 5060 | GB28181 视频监控 |

### 第三步～第五步：AI / VIDEO / WEB

```bash
cd AI    && ./install_linux.sh install
cd VIDEO && ./install_linux.sh install
cd WEB   && ./install_linux.sh install
cd APP   && ./install_linux.sh install   # 仅 full
```

### 仅部署业务模块（不含中间件）

```bash
cd .scripts/docker
./install_business_linux.sh install              # 全部业务模块
./install_business_linux.sh update DEVICE WEB    # 指定模块更新
./install_business_linux.sh verify
```

## 常用运维命令

### 统一脚本（全平台）

```bash
cd .scripts/docker   # 或在项目根用 .scripts/docker/install_linux.sh

./install_linux.sh install    # 首次安装
./install_linux.sh start      # 启动
./install_linux.sh stop       # 停止
./install_linux.sh restart    # 重启
./install_linux.sh status     # 状态
./install_linux.sh logs       # 全部日志（最近 100 行）
./install_linux.sh logs WEB   # 指定模块日志
./install_linux.sh build      # 本地重新构建镜像
./install_linux.sh update     # 更新并重启（可选拉取/重建）
./install_linux.sh verify     # 健康验证
./install_linux.sh check      # 检查 Docker 环境
./install_linux.sh profile    # 查看部署规格
./install_linux.sh clean      # 清理容器与镜像（危险）
./install_linux.sh pull       # 拉取预构建运行时镜像
./install_linux.sh help       # 帮助
```

### 单模块脚本

每个模块目录（`DEVICE` / `AI` / `VIDEO` / `WEB` / `APP`）均支持：

```bash
./install_linux.sh install | start | stop | restart | status | logs | build | clean | update
```

中间件单独管理：

```bash
cd .scripts/docker
./install_middleware_linux.sh install | start | stop | restart | status | logs | build | clean | update
```

### 常用环境变量

| 变量 | 说明 |
|------|------|
| `EASYAIOT_DEPLOY_PROFILE` | 部署规格：`mini` / `standard` / `full` |
| `HOST_IP` | 强制指定宿主机 IP，跳过自动探测 |
| `PARALLEL_MODULES=true` | 业务模块并行启动/更新（内存充足时） |
| `PARALLEL_BUILD=true` | 并行构建（默认串行，防 OOM） |
| `FORCE_NETWORK_RECREATE=true` | 宿主机 IP 变更后重建 Docker 网络 |
| `EASYAIOT_RUNTIME_REGISTRY` | 预构建镜像仓库地址 |

## 预构建镜像（可选）

项目支持从远程仓库拉取预构建业务镜像，跳过耗时的本地 Maven / pnpm / pip 构建。

配置文件：`.scripts/docker/runtime_registry.conf`

```bash
# 交互式拉取（安装前或 update 时选择）
.scripts/docker/install_linux.sh pull

# 构建并推送运行时镜像（CI/发布用）
.scripts/docker/install_linux.sh build-runtime          # 全部模块
.scripts/docker/install_linux.sh build-runtime DEVICE   # 指定模块
```

拉取成功后，后续 `install` / `update` 会检测到 `.runtime_images_pulled` 标记并直接启动容器。

## GPU 配置

### 安装与验证

```bash
nvidia-smi

# 安装 NVIDIA Container Toolkit
# https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html
```

### 自动检测

安装脚本会自动检测 GPU：

- 有 GPU → 启用 `runtime: nvidia`、`NVIDIA_VISIBLE_DEVICES=all`
- 无 GPU → CPU 模式运行

### 多 GPU

```bash
export CUDA_VISIBLE_DEVICES=0,1
```

## 故障排查

### 服务启动失败

```bash
docker ps -a
docker logs -f postgres-server
docker logs -f nacos-server
docker logs -f ai-service
docker logs -f video-service
.scripts/docker/install_linux.sh logs
```

### 网络问题

```bash
docker network ls | grep easyaiot
docker network inspect easyaiot-network

# 宿主机 IP 变更后
export FORCE_NETWORK_RECREATE=true
.scripts/docker/install_linux.sh restart
```

### PostgreSQL / Redis

```bash
.scripts/docker/fix_postgresql.sh
.scripts/docker/fix_redis.sh
```

### Docker 系统问题

```bash
sudo .scripts/docker/diagnose_docker_systemd.sh diagnose
sudo .scripts/docker/diagnose_docker_systemd.sh fix-all
.scripts/docker/cleanup_docker_space.sh
df -h && docker system df
```

### Kafka 消费组

```bash
cd VIDEO && ./fix_kafka_consumer_group.sh
```

### 端口冲突

修改对应模块 `docker-compose.yml` 中的端口映射，或停止占用进程。

### 切换部署规格后 WEB 异常

前端编译时写入了部署规格，切换后需重建 WEB：

```bash
cd WEB && ./install_linux.sh build
```
