# GB28181 国标设备接入说明

本文档说明如何将 GB/T 28181 国标设备（摄像头、NVR、平台等）接入本系统的 iot-gb28181 服务。接入前请根据实际部署的 SIP 服务配置填写设备端参数。

---

## 一、平台 SIP 服务配置说明

本系统（iot-gb28181）作为 **SIP 服务器**（国标上级平台），设备作为 **SIP 客户端** 向平台注册。平台侧配置在以下文件中（按环境选择）：

| 环境 | 配置文件 |
|------|----------|
| 默认/公共 | `iot-gb28181-biz/src/main/resources/application.yaml` |
| 本地 | `application-local.yaml` |
| 开发 | `application-dev.yaml` |
| 生产 | `application-prod.yaml` |

### 1.1 关键配置项（sip 节点）

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `sip.port` | 平台 SIP 服务监听端口 | `8116` |
| `sip.domain` | 国标域（ID 前 10 位，见国标附录 D） | `4101050000` |
| `sip.id` | 平台国标 ID（SIP 服务器 ID） | `41010500002000000001` |
| `sip.password` | 默认设备认证密码（可选；不配置则需预先在平台添加设备并设密） | 如不配置则必须预置设备 |
| `sip.register-password-auth` | 是否启用设备 REGISTER 密码鉴权（关闭后跳过 401 挑战与密码校验） | `true`（默认） |
| `sip.ip` | 本机监听 IP，多机用逗号分隔；不配则自动绑定所有非 docker 网卡 | `192.168.240.200` 或 `192.168.240.200,192.168.240.201` |
| `sip.alarm` | 是否存储告警 | `true` / `false` |

- **多机部署**：若有多台服务器同时提供 SIP 服务，`sip.ip` 可配置为逗号分隔的多个 IP（如 `192.168.240.200,192.168.240.201`），设备端可任选其一或配置主备。
- **设备认证**：若配置了 `sip.password`，设备可使用该公共密码注册；否则需在平台先添加设备并设置设备独立密码，设备使用该密码注册。
- **兼容无鉴权设备**：若设备不支持 REGISTER 鉴权（部分海康型号），可将 `sip.register-password-auth` 设为 `false`；建议仅在受控内网环境使用。

接入设备时，设备端填写的 **SIP 服务器地址、端口、域、ID、认证密码** 必须与上述平台配置一致（或与平台已预置的设备信息一致）。

## 二、设备端接入参数说明

在摄像头/NVR/国标客户端等设备上配置“国标上级平台”或“SIP 服务器”时，请按下面含义填写（名称因厂商不同可能略有差异）。

| 设备端参数名（常见） | 说明 | 填写规则 |
|----------------------|------|----------|
| 传输协议 | SIP 信令传输协议 | 选 **UDP** 或 **TCP**（与平台支持一致，一般 UDP 即可） |
| 协议版本 | 国标版本 | **GB/T 28181-2016** 或 **GB/T 28181-2022** |
| SIP 服务器 ID | 平台国标 ID | 与平台 `sip.id` 一致，如 `41010500002000000001` |
| SIP 服务器域 | 平台域 | 与平台 `sip.domain` 一致，如 `4101050000` |
| SIP 服务器地址 | 平台 SIP 服务 IP | 平台 `sip.ip` 中的任一个（多机时填主用或备用 IP） |
| SIP 服务器端口 | 平台 SIP 端口 | 与平台 `sip.port` 一致，如 `8116` |
| SIP 用户名 / 设备国标 ID | 本设备国标编号 | 20 位国标编码，平台内唯一，如 `34020000001320000004` |
| SIP 用户认证 ID | 认证用 ID | 一般与“SIP 用户名”相同 |
| SIP 用户认证密码 | 设备注册密码 | 平台公共密码或该设备在平台预置的密码 |
| 本地 SIP 端口 | 设备本机 SIP 端口 | 设备本机监听端口，如 `5060`（由设备或网络环境决定） |

- **设备国标 ID**：需在平台内唯一；若平台未开启公共密码，需先在平台“国标设备”中预添加该 ID 并设置密码，设备端认证密码填该密码。
- 实际部署时，请以 **当前运行环境的 `application-*.yaml` 中 sip 配置** 为准，替换下文示例中的地址、端口、域、ID、密码。

## 四、如何与当前部署的 SIP 配置对齐

1. **确认运行环境**：确定当前使用的是 local/dev/prod 等哪种 profile。  
2. **查看对应配置文件**：打开该环境下的 `application-<profile>.yaml`（或默认 `application.yaml`）。  
3. **核对 sip 节点**：  
   - `sip.id` → 设备端“SIP 服务器 ID”  
   - `sip.domain` → 设备端“SIP 服务器域”  
   - `sip.port` → 设备端“SIP 服务器端口”  
   - `sip.ip`（或实际绑定的网卡 IP）→ 设备端“SIP 服务器地址”（多机时取对应机器的 IP）  
   - `sip.password` 或 预置设备密码 → 设备端“SIP 用户认证密码”  
4. **设备 ID**：设备端“SIP 用户名/设备国标 ID”在平台内唯一；若未使用公共密码，需先在平台添加该设备并设置密码。  

使用本项目提供的 **设备接入信息生成脚本** 时，脚本会生成多组设备参数；请将脚本中的 **SIP 服务器地址、端口、域、ID、默认密码** 等替换为上述配置中的实际值，再批量填写到设备或导入到配置表中。

## 五、批量生成设备接入信息

项目提供脚本用于批量生成设备接入参数（设备 ID、密码等），便于批量录入或下发到设备。

- **脚本路径**：`DEVICE/iot-gb28181/scripts/gen_gb28181_device_access_info.sh`  
- **用法**：  
  - 默认生成 **10 组** 设备接入信息：  
    `./gen_gb28181_device_access_info.sh`  
  - 指定数量（1～100）：  
    `./gen_gb28181_device_access_info.sh 25`  
- 生成内容包含：设备国标 ID、认证密码、以及上述表格中的各 SIP 参数（可配置为与当前部署的 sip 一致）。  
- **配置方式**：  
  - 直接修改脚本内「可配置项」默认值（SIP 服务器 ID/域/端口/地址、设备 ID 前缀、本地端口等）；  
  - 或通过环境变量覆盖，例如：  
    `SIP_SERVER_ADDRS=192.168.1.100 SIP_SERVER_PORT=8116 ./gen_gb28181_device_access_info.sh 10`  

详细可配置项见脚本内注释。

## 六、常见问题

1. **设备注册 401/403**  
   - 401：未带认证或认证参数错误，请检查设备端“SIP 用户认证 ID”和“SIP 用户认证密码”是否与平台一致（公共密码或该设备预置密码）。  
   - 403：密码错误或平台未配置公共密码且未预置该设备，请核对密码或在平台添加该设备并设密。  

2. **设备离线或无法连接**  
   - 检查设备端“SIP 服务器地址”“SIP 服务器端口”是否与平台实际监听 IP 和 `sip.port` 一致。  
   - 若平台有多机，确认设备访问的 IP 是否为当前提供 SIP 服务的那台机器。  
   - 检查防火墙、安全组是否放行 UDP/TCP 的 SIP 端口（如 8116）。  

3. **多台服务器如何填**  
   - 设备端一般只能填 1～2 个“SIP 服务器地址”（主/备）。  
   - 填写为当前实际提供 SIP 服务的机器 IP（即 `sip.ip` 中的某一个），端口均为 `sip.port`。  

4. **国标 ID 与域**  
   - 设备国标 ID 为 20 位，平台内唯一；域为平台 `sip.domain`（通常为 ID 前 10 位），设备端“SIP 服务器域”必须与平台一致。
